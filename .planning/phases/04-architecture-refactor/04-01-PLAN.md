   - Set dialogue to `"The way is clear!"`
   - After `GAME_CONFIG.timing.pathClearedToDoorsOpenMs` (800ms): set doorOpen true
   - After additional `GAME_CONFIG.timing.doorOpenToWalkStartMs` (600ms): set isWalking true, hide dialogue
   - Set phase to `{ phase: 'transitioning' }`

4. `handleWalkComplete` callback — from lines 566-572. Sets isWalking false, pandaGone true, dialogue to `"A new day begins."`, shows dialogue, sets phase to `{ phase: 'check-in' }`

5. Pan gesture — from lines 615-642. `Gesture.Pan().onUpdate()` with hit detection for fog wisps (using `sceneConfig.FOG_WISPS`, `fogTouchRadiusBuffer`) and leaves (using `sceneConfig.LEAVES`, `leafTouchRadius`). `.minDistance(0)`.

6. Fog persistence — from lines 607-612. `useEffect` calling `saveFogState([...clearedFog], [...clearedLeaves])` when sets change. No-op in reset mode.

7. Mount-time fog restore — on mount, call `loadFogState()`. If result exists, populate clearedFog and clearedLeaves Sets.

8. `resetScene()` function — from lines 703-716 (the parts relevant to scene). Resets clearedFog, clearedLeaves to empty Sets, doorOpen/isWalking/pandaGone to false.

**Per D-07 (indirectly):** This hook does NOT call `loadWorldState()`. The check for `lastVisitDate === today` (returning user detection) happens in the screen's mount effect, not in this hook.
